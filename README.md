# Vocab-Swipe 🇹🇼 中文詞彙學習

A modern, interactive Traditional Chinese vocabulary learning web application built with vanilla JavaScript. Learn Chinese characters and vocabulary through engaging card-swipe gameplay with multiple learning modes.

## Features

- **Multiple Learning Modes**
  - Classic vocabulary mode (`index-classic.html`)
  - Stroke order learning (`index-stroke.html`)
  - Character/token learning (`index-token.html`)
  - Wallpaper-themed mode (`index-wallpaper.html`)

- **Interactive Gameplay**
  - Swipe-based card interaction
  - Pair-matching quizzes in rounds of 30
  - Example sentences with token highlighting
  - Stroke animation and visualization

- **Rich Learning Data**
  - Comprehensive vocabulary database with English translations
  - Supplemental token/character definitions
  - Contextual example sentences
  - Offline reference for Hanzi animations

- **Visual Polish**
  - Rotating wallpaper backgrounds
  - Responsive card-based interface
  - Traditional Chinese localization

## Quick Start

### Prerequisites
- Modern web browser (Chrome, Firefox, Safari, Edge)
- Python 3.6+ (for wallpaper sync utility)

### Running the Application

1. Open the main application in your browser:
   ```
   index.html
   ```

   Or access alternate learning modes:
   - `index-classic.html` - Classic mode
   - `index-stroke.html` - Stroke learning
   - `index-token.html` - Token learning
   - `index-wallpaper.html` - Wallpaper background mode

2. Click "Start Game" to begin learning

## Development

### Modifying Vocabulary

The vocabulary is stored in `vocab-data.json`. To add or modify words:

1. Edit `vocab-data.json`
2. Add entries to the `data` array with the following structure:
   ```json
   {
     "zh": "詞彙",
     "en": "vocabulary, lexicon",
     "example": "這本詞典是個有用的詞彙資源。"
   }
   ```
3. Reload the browser (the app uses `cache: 'no-store'` for fresh data)

### Managing Wallpapers

Wallpapers are stored in the `wallpapers/` directory and must be indexed in `wallpapers.json`:

1. Add image files (JPG/PNG) to `wallpapers/` directory
2. Run the sync utility:
   ```bash
   python sync_wallpapers_json.py
   ```
   Or with custom paths:
   ```bash
   python sync_wallpapers_json.py --dir custom/wallpapers --out custom/manifest.json
   ```
3. Commit both the images and the updated `wallpapers.json`

**Note:** Do not manually edit `wallpapers.json` — it is auto-generated by the sync script.

### Modifying Game Logic

The application uses vanilla JavaScript embedded in HTML files:

1. Locate the `<script>` block in the desired HTML file
2. Edit the JavaScript directly
3. Reload your browser to test changes
4. For cross-mode changes, update all relevant `index-*.html` files

### Key Code Sections

- **Dictionary System**: `DICT` object with case-insensitive lookups
- **Tokenization**: `tokenizeChinese()` function for character segmentation
- **Game Mechanics**: 
  - `ROUND_SIZE = 30` — standard quiz round size
  - `PAIR_SKIP` — character pairs to exclude from matching
  - `PRONOUN_ALIASES` — linguistic variant mapping
- **Asset Loading**: `assetUrl()` helper for relative path resolution
- **Data Fetching**: Uses `cache: 'no-cache'` for wallpapers, `cache: 'no-store'` for vocabulary

## Testing

There is no automated test suite. Manual testing should verify:

- Each game mode variant loads and functions correctly
- Vocabulary displays with correct Chinese and English text
- Wallpapers load and rotate properly
- Example sentences display and tokenize correctly
- Swipe interactions work as expected

Test wallpaper sync:
```bash
python sync_wallpapers_json.py
```

Verify output shows the correct number of files and only updates when the manifest changes.

## Architecture Notes

### Data-Driven Design
The application is built around JSON data files:
- **vocab-data.json**: Contains vocabulary with translations and examples
- **wallpapers.json**: Auto-generated manifest of wallpaper filenames

### No Build System
This project uses vanilla JavaScript with no bundler (Webpack, Vite, etc.). All code is embedded directly in HTML files for simplicity and portability.

### Relative Asset Loading
The app uses relative URLs based on the document location, enabling it to run from any directory path. The `assetUrl()` helper resolves paths relative to the app's base URL.

### Chinese Language Localization
- **Default**: Traditional Chinese (繁體中文)
- **Supported**: Simplified Chinese example sentences
- **Punctuation**: Handles both English and Chinese-specific marks (，。！？；：、)
- **Tokenization**: Respects Chinese character boundaries and dictionary tokens
